# PhD Assistant Knowledge Base

A structured repository for organizing PhD coursework materials, assignments, papers, and resources for Information Technology doctoral program (Fall 2025 - ).

## Purpose

This repository serves as a knowledge base to help Claude Code assist you with:
- Writing papers and assignments
- Understanding course requirements
- Managing multiple courses per semester
- Organizing research and academic materials

## Quick Start

### 1. Download Materials from Blackboard

For each course, download:
- Course syllabus
- Weekly assignments and instructions
- Quizzes (save in assignments folder)
- Announcements
- Grades/feedback
- Messages from instructor

### 2. Organize in Course Folders

Place materials in the appropriate course folder under the current semester:

```
semesters/2025-fall/[COURSE-CODE-Course-Name]/[category]/
```

### 3. Use with Claude Code

When you need help with an assignment or paper:
1. Ensure relevant course materials are in the repository
2. Ask Claude Code for assistance
3. Claude will reference your course materials for context

### Course Folder Structure

Each course contains:
- `syllabus/` - Course syllabus and policies
- `assignments/` - All assignments including quizzes
- `announcements/` - Course announcements
- `grades/` - Grade tracking
- `messages/` - Instructor communications

## File Naming Conventions

Use clear, descriptive names:
- `syllabus.pdf` or `syllabus.docx`
- `week-1-assignment.docx`
- `quiz-1-instructions.pdf`
- `2025-09-15-announcement.md`
- `midterm-grades.csv`

Include dates or week numbers where relevant.

## Supported File Formats

- **PDF** (.pdf) - Syllabi, readings, assignments
- **Word** (.docx) - Editable documents, papers
- **Excel** (.xlsx) - Grade tracking, data
- **CSV** (.csv) - Simple tables, grades
- **Markdown** (.md) - Notes, summaries

## Tips for Organization

1. **Download regularly** - Get materials from Blackboard weekly
2. **Maintain structure** - Keep files in designated folders
3. **Note due dates** - Include deadlines in filenames or tracking file
4. **Version papers** - Use git commits when working on papers
5. **Back up regularly** - Push to GitHub/remote regularly

## Privacy & Security

- Be cautious about committing sensitive information (grades, personal data)
- Consider making repository private if it contains personal information
- Use `.gitignore` to exclude sensitive files
