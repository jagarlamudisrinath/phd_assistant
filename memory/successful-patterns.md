## Update Protocol

### After Each High-Scoring Assignment (95%+)

1. **Analyze what worked**:
   - Read professor praise carefully
   - Identify specific elements highlighted
   - Review your approach and structure
   - Note timing and workflow

2. **Document the pattern**:
   - Add to appropriate section above
   - Include concrete example
   - Note grade and professor feedback
   - Provide replication steps

3. **Abstract to general principle**:
   - Is this assignment-specific or broadly applicable?
   - What's the underlying success factor?
   - How can JARVIS replicate this?

### After Each Lower-Scoring Assignment (<90%)

1. **Identify gaps**:
   - What was missing compared to high-scoring work?
   - What did professor flag as issue?

2. **Update anti-patterns**:
   - Document what didn't work
   - Provide corrective approach

### Quarterly Pattern Review

**Look for**:
- Patterns that appear across multiple assignments
- Course-specific vs. universal patterns
- Emerging trends in professor feedback
- Adjustments to make in future work

## JARVIS Calibration Notes

### Using Successful Patterns

When JARVIS drafts assignments, it should:

1. **Scan this file** for patterns matching the current assignment type
2. **Apply relevant patterns**: Structure, evidence integration, analysis depth
3. **Adapt, don't copy**: Use pattern as guide, not template (avoid self-plagiarism)
4. **Combine patterns**: Mix multiple successful approaches where appropriate
5. **Note in comments**: Which patterns are being applied (for transparency)

### Pattern Confidence Levels

- **High confidence** (3+ successful uses): Apply liberally
- **Medium confidence** (1-2 uses): Apply with caution
- **Emerging pattern** (1 use): Note but don't over-rely on

### When Patterns Conflict

If multiple patterns apply but contradict:
1. Prioritize most recent pattern (professors may update expectations)
2. Prioritize course-specific pattern over general
3. Check professor preferences file for tiebreaker
4. Ask user for guidance
