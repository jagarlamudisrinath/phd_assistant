## Learning from Feedback

### Feedback-to-Action Pipeline

**Step 1: Categorize Feedback**
- Positive reinforcement → Add to strengths
- Constructive criticism → Add to improvements
- Specific error → Add to issues list

**Step 2: Prioritize**
- Frequent feedback → High priority
- Course-specific → Medium priority
- One-off comments → Low priority (may be assignment-specific)

**Step 3: Develop Action Plan**
- What to change in next assignment
- Specific technique to implement
- Resource to consult if needed

**Step 4: Implement & Verify**
- Apply change in next similar assignment
- Check if professor feedback changes
- Confirm improvement recognized

**Step 5: Update This File**
- Mark as addressed
- Document evidence of improvement
- Update status tracking

## JARVIS Integration

### How JARVIS Uses This File

**During Drafting** (academic-paper-writer):
- **Check recurring issues**: Avoid common mistakes proactively
- **Apply recurring strengths**: Emphasize elements that are consistently praised
- **Address flagged areas**: Specifically target improvement areas

**During Review** (phd-assignment-reviewer):
- **Flag known issues**: "This area has been flagged before. Verify it's addressed."
- **Confirm strengths**: "Maintain the [strength] that professors praise."
- **Compare to patterns**: "Does this match patterns that score well?"

**During Revision** (revise-submission):
- **Cross-reference feedback**: "Is this similar to past feedback? Apply known fix."
- **Prevent regression**: "Don't reintroduce issues that were previously corrected."

## Red Flags (Stop and Fix)

### Critical Issues That Must Be Fixed

If any of these appear 3+ times in feedback:
- **Citation errors** → Review APA guide, use citation manager
- **Weak analysis** → Study critical thinking frameworks
- **Poor organization** → Review paper structure best practices
- **Mechanical errors** → Use Grammarly, proofread multiple times

**Escalation**: If issue appears 3+ times, make it top priority for next assignment.

## Update Protocol

### After Each Graded Assignment

1. **Extract all feedback**:
   - Read professor comments thoroughly
   - Note inline comments, rubric scores, and summary feedback
   - Categorize: positive, constructive, specific issues

2. **Check for patterns**:
   - Is this new feedback or recurring?
   - Similar to past comments?
   - Course-specific or universal?

3. **Update this file**:
   - Add to recurring sections if pattern (2+ occurrences)
   - Update frequency counts
   - Note action taken or planned

4. **Trigger improvements**:
   - If recurring issue, develop specific action plan
   - If new strength, document for continuation
   - If critical issue (3+ times), escalate priority

### Monthly Review

**Analyze trends**:
- Are improvement areas showing progress?
- Are strengths being maintained?
- New patterns emerging?

**Adjust priorities**:
- Reprioritize based on recurring vs. resolved issues
- Update action plans as needed

### Semester Review

**Comprehensive assessment**:
- Compare early vs. late feedback
- Document improvement trajectory
- Identify next semester focus areas
- Archive resolved issues

## Initial Calibration

**First 3 assignments**:
- Collect baseline feedback
- Identify initial patterns
- Don't over-react to single instances

**Assignments 4-10**:
- Patterns should emerge clearly
- Focus on frequent feedback
- Implement targeted improvements

**Beyond 10 assignments**:
- Mature pattern recognition
- Proactive issue prevention
- Fine-tuning of strengths
- Minimal recurring issues

**Goal**: By end of first semester, have clear understanding of strengths to maintain and issues to avoid.
